一、版本控制管理工具
  (一)集中式版本控制管理
    如CVS,有一个集中管理的服务器,可以把多人分工完成的内容合并
  (二)分布式版本控制管理
    如git,最初是为了管理Linux内核代码而诞生的
二、git
  (一)Sheel指令
    1.打印当前工作路径,全称:print working directory
      pwd
    2.切换路径,全称:change directory,可以选择相对路径和绝对路径
      (1)路径为..可以切换到上级目录,路径主要写a/b/c/d和/c
        cd 路径
      (2)cd ~切换到主目录
        cd ~
    3.显示当前目录中的所有非隐藏的文件或目录,全称:list,文件名前加.可以隐藏
      ls
      (1)查看最后访问时间
        ls -lt
      (2)显示当前目录中的所有文件或目录,包括隐藏文件
        ls -a
  (二)对文件和文件夹的操作
    1.新建文件夹,全称:make directory,可以选择相对路径和绝对路径
      (1)在当前路径下新建文件夹
        mkdir 新文件夹名
      (2)在已有文件夹下创建新文件夹,若已有文件夹不存在会报错
        mkdir 已有文件夹名/新文件夹名
      (3)在已有文件夹下创建新文件夹,若已有文件夹不存在会一同创建
        mkdir -p 已有文件夹名/新文件夹名
    2.新建文件,全称:touch,可以选择相对路径和绝对路径
      (1)在当前路径下新建文件,若文件已存在会修改其最后访问时间
        touch 文件名.文件类型
      (2)在已有文件夹下创建新文件,若文件已存在会修改其最后访问时间
        touch 已有文件夹名/文件名.文件类型
    3.删除空文件夹,全称:remove directory,可以选择相对路径和绝对路径
      (1)在当前路径下删除空文件夹
        rmdir 空文件夹名
      (2)在已有文件夹下删除空文件夹,只会删除最后一层的空文件夹
        rmdir 已有文件夹名/已有文件夹名/空文件夹名
    4.删除文件或文件夹及其内包含的所有,全称:remove,可以选择相对路径和绝对路径
      rm -rf 文件名|文件夹名
